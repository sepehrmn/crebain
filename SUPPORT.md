# Support

Thanks for using CREBAIN. Here is where to go for each kind of help.

## Questions and discussion

For usage questions, setup help, or to discuss an idea before filing an issue,
use [GitHub Discussions](https://github.com/sepehrmn/crebain/discussions).

## Bugs and feature requests

Open an [issue](https://github.com/sepehrmn/crebain/issues/new/choose) using the
bug-report or feature-request form. Include the OS/hardware, app mode,
commit/version, and any backend/model/ROS context, as the forms prompt.

## Security vulnerabilities

Do **not** open a public issue. Follow the process in
[SECURITY.md](SECURITY.md) (GitHub private vulnerability reporting).

## Before asking

- Read the [README](README.md) and, for contributing, [CONTRIBUTING.md](CONTRIBUTING.md).
- Run `bun run validate` (frontend) or `bun run validate:all` (full) to confirm
  whether the problem reproduces against a clean checkout.
- This is a research prototype: model weights are not shipped, and performance,
  accuracy, and transport-latency numbers must be measured on your own target
  hardware (see the disclaimers in the README and `SECURITY.md`).
